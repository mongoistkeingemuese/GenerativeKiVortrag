Was ist Generative KI?

  • KI, die neue Inhalte erzeugt (Text, Code, Bilder)
  • Trainiert auf riesigen Datensätzen
  • Lernt Muster und Zusammenhänge
  • Generiert Ausgaben basierend auf Eingaben (Prompts)

Kernidee

Statt Regeln zu programmieren, lernt das Modell aus Beispielen — und kann dann neue, ähnliche Inhalte produzieren.

Generative KI unterscheidet sich von klassischer KI: Sie erstellt neue Inhalte statt nur zu klassifizieren oder vorherzusagen. Für uns Entwickler besonders relevant: Code-Generierung.

LLMs im Überblick

  • GPT-4o / o1 — OpenAI, Allrounder mit starker Code-Fähigkeit
  • Claude — Anthropic, exzellent für längere Kontexte und Analyse
  • Gemini — Google, multimodal (Text, Bild, Code)
  • Llama / Mistral — Open-Source-Alternativen, lokal betreibbar
  • CodeLlama / DeepSeek — Spezialisiert auf Code

Alle Modelle werden ständig besser. Entscheidend ist nicht welches Modell, sondern wie man es einsetzt.

Prompt Engineering Grundlagen

  • Sei spezifisch — Je genauer der Prompt, desto besser das Ergebnis
  • Kontext mitgeben — Sprache, Framework, Constraints
  • Beispiele liefern — Ein-/Ausgabe-Paare zeigen das gewünschte Format
  • Rolle definieren — "Du bist ein Senior Python-Entwickler"
  • Iterativ arbeiten — Ergebnis verfeinern statt alles auf einmal
Prompt Engineering ist die Schlüsselkompetenz. Nicht das Modell macht den Unterschied, sondern wie gut man promptet.

Dos and Don'ts

Do

  • KI-Code immer reviewen
  • Als Startpunkt nutzen, nicht als Endprodukt
  • Tests schreiben für generierten Code
  • Prompt-Muster dokumentieren und teilen

Don't

  • Blind Copy-Paste ohne Verständnis
  • Sensible Daten in Cloud-Prompts
  • KI-Code als 100% korrekt annehmen
  • Lizenzen ignorieren bei generiertem Code
Wichtige Regeln für den verantwortungsvollen Einsatz. Der Punkt mit sensiblen Daten ist besonders wichtig für Unternehmenskontexte.

Sicherheit & Datenschutz

  • Keine Secrets in Prompts — API-Keys, Passwörter, Tokens
  • Firmenrichtlinien beachten — Welche Tools sind freigegeben?
  • Lokale Alternativen — Self-hosted Modelle für sensible Projekte
  • Code-Review bleibt Pflicht — KI kann Sicherheitslücken einbauen
  • DSGVO & Compliance — Wohin fließen die Daten?

Faustregel: Was du nicht in eine öffentliche E-Mail schreiben würdest, gehört nicht in einen Cloud-Prompt.

Sicherheit ist kein Blocker, aber muss bedacht werden. Enterprise-Lösungen wie GitHub Copilot Business oder Azure OpenAI bieten mehr Kontrolle.

Grenzen von KI-Assistenten

  • Halluzinationen — KI erfindet plausibel klingende Falschinformationen
  • Veraltetes Wissen — Training-Cutoff, kennt neueste APIs nicht
  • Kontextfenster — Große Codebasen passen nicht komplett rein
  • Kein echtes Verständnis — Mustererkennung, nicht Logik
  • Architektur-Entscheidungen — Hier bleibt der Mensch gefragt
Ehrlich über Grenzen sprechen schärft die Glaubwürdigkeit. KI ist ein Werkzeug, kein Ersatz für Denken.

Integration in den Workflow

  • IDE-Integration — Copilot/Continue direkt im Editor
  • CLI-Tools — Claude Code, Aider für Terminal-Workflows
  • Code-Review — KI als erster Reviewer vor dem PR
  • CI/CD — Automatische Test-Generierung in der Pipeline
  • Dokumentation — README, API-Docs, Changelogs generieren

Die beste Integration ist die, die man nicht merkt — nahtlos im bestehenden Workflow.

Zeige konkrete Workflows: Git commit message generieren, PR-Beschreibung, Changelog. Die KI passt sich dem Workflow an, nicht umgekehrt.
