Anwendungsarten

Vier Wege, generative KI im Alltag einzusetzen:

💬 Chat

  • Web-Oberfläche im Browser
  • Frage-Antwort, Brainstorming, Erklärungen
  • Niedrigste Einstiegshürde

Tools: ChatGPT, Claude, Gemini, Le Chat

💻 CLI

  • KI direkt im Terminal / in der IDE
  • Liest & schreibt Dateien, führt Befehle aus
  • Arbeitet autonom an komplexen Aufgaben

Tools: Claude Code, GitHub Copilot, Cursor, Aider

🔌 API

  • Programmatischer Zugriff per REST/SDK
  • Integration in eigene Anwendungen & Pipelines
  • Volle Kontrolle, skalierbar, automatisierbar

Anbieter: OpenAI, Anthropic, Google, Mistral

🤖 Agentic Systems

  • Autonome KI-Agenten mit Tool-Zugriff
  • Planen, entscheiden & handeln eigenständig
  • Orchestrierung mehrerer Agenten möglich

Beispiele: OpenAI Agents SDK, Claude Agent SDK, CrewAI

Chat ist der einfachste Einstieg. CLI-Tools sind der Produktivitäts-Boost für Entwickler. API ist für eigene Produkte und Automatisierung. Agentic Systems sind die nächste Stufe: KI-Agenten, die eigenständig planen, Tools nutzen und Aufgaben lösen. Viele nutzen mehrere Ansätze parallel.

Der Anfang: 20€ und ein Tetris-Spiel

  • Google AI Premium Abo — 20€/Monat, Gemini CLI installiert
  • Erster Test: „Bau mir Tetris in JavaScript“
  • Ein Prompt, ein spielbares Ergebnis — innerhalb von Minuten
  • Wow-Moment: Das funktioniert wirklich?!
  • Sofort nächste Idee: Was geht noch alles?

Von null auf spielbares Tetris in unter 5 Minuten — mit einem einzigen Prompt.

Der Anfang war pure Begeisterung. Gemini CLI installiert, einen Satz getippt, und plötzlich hatte ich ein funktionierendes Tetris. Das war der Moment, wo klar wurde: Das ist kein Spielzeug.

Dann kam die Ernüchterung

  • Rate Limits — Google drosselt, Abbrüche mitten im Flow
  • Kontextverlust — Modell vergisst bei längeren Sessions
  • Inkonsistenz — Gleicher Prompt, anderes Ergebnis
  • Überschreibungen — Funktionierender Code wird „verbessert“ und bricht
  • Erkenntnis: Einfach prompten und hoffen reicht nicht

Vibe Coding hat eine Grenze — und die erreicht man schneller als gedacht.

Nach dem Wow-Moment kam die Realität. Projekte wurden größer, Probleme häuften sich. Die KI hat Code überschrieben, Kontext vergessen, und Rate Limits haben den Flow zerstört. Der Punkt, an dem viele aufgeben — oder lernen.

Grenzen von KI-Assistenten

  • Halluzinationen — KI erfindet plausibel klingende Falschinformationen
  • Veraltetes Wissen — Training-Cutoff, kennt neueste APIs nicht
  • Kontextfenster — Große Codebasen passen nicht komplett rein
  • Kein echtes Verständnis — Mustererkennung, nicht Logik
  • Architektur-Entscheidungen — Hier bleibt der Mensch gefragt

Wer die Grenzen kennt, setzt KI gezielter ein — und bleibt dort Mensch, wo es darauf ankommt.

Ehrlich über Grenzen sprechen schärft die Glaubwürdigkeit. KI ist ein Werkzeug, kein Ersatz für Denken. Diese Grenzen zu kennen ist der erste Schritt, sie zu umgehen.

Die Evolution: Drei Stufen

1

Vibe Coding

Prompten und hoffen. Unvorhersehbare Qualität, kein Kontext, viel Nacharbeit. Funktioniert für Prototypen und kleine Scripts.

2

Context Engineering

Projektregeln, Architektur-Docs und Konventionen bereitstellen. Die KI „kennt“ das Projekt — konsistenter Code, weniger Nacharbeit.

3

Agentic Systems

KI plant, implementiert und testet selbstständig. Entwickler reviewt und steuert. Mehrere Agenten mit spezialisierten Rollen.

Der größte Sprung liegt zwischen Stufe 1 und 2 — Context Engineering ist der Schlüssel.

Drei Stufen statt sechs, weil die wesentlichen Sprünge hier liegen. Vibe Coding ist, wo die meisten starten. Context Engineering ist, wo die Produktivität explodiert. Agentic Systems ist die Zukunft.

Context Engineering im Detail

Was man bereitstellt

  • CLAUDE.md / Rules — Projektregeln, Konventionen, Do's & Don'ts
  • Architektur-Docs — Dateistruktur, Patterns, Abhängigkeiten
  • Beispielcode — So sieht guter Code in diesem Projekt aus
  • Test-Patterns — Wie wird hier getestet?

Was man bekommt

  • Konsistenter Code — Passt zum bestehenden Stil
  • Weniger Nacharbeit — KI hält sich an Regeln
  • Bessere Architektur — Entscheidungen im Kontext
  • Reproduzierbare Qualität — Unabhängig von Session-Länge

Context Engineering ist keine Raketenwissenschaft — es ist eine Markdown-Datei, die den Unterschied macht.

Das war mein Durchbruch-Moment. Statt immer wieder die gleichen Korrekturen zu machen, einmal die Regeln aufschreiben. CLAUDE.md für Claude Code, .cursorrules für Cursor, AGENTS.md für Codex. Einmalige Investition, dauerhafter Nutzen.

Automatisiert lösen mit KI

Von manuell prompten zu autonomen Agenten:

  • Problem zerlegen — Kleine, klare Aufgaben — ein Agent pro Teilproblem
  • Kontext bereitstellen — Architektur, Constraints, Beispiele — Agenten lesen CLAUDE.md & Co.
  • Automatisch iterieren — Agent testet, erkennt Fehler, korrigiert selbstständig
  • Deterministisch absichern — Tests, Linter, Type-Checks als automatische Guardrails für Agenten
  • Review-Agenten einsetzen — Separater Agent mit eigenem Kontext prüft Output
  • Orchestrieren statt prompten — Mehrere spezialisierte Agenten statt ein Prompt für alles

Agentic AI verschiebt die Rolle: vom Prompten zum Orchestrieren autonomer Systeme — du definierst das Ziel, Agenten finden den Weg.

Der Shift von „Probleme lösen“ zu „automatisiert lösen“ ist der Kern von Agentic AI. Statt einzelne Prompts abzusetzen, definiert man Ziele und lässt spezialisierte Agenten autonom arbeiten. Die Guardrails — Tests, Linter, Reviews — werden zu den Leitplanken, innerhalb derer Agenten frei agieren.

Projekt: PLC_Express (PLCX)

Problem

  • OPC UA — zu langsam für Echtzeit-Daten
  • Komplexität — Overhead für einfache Anwendungen
  • Latenz — Millisekunden zählen in der Automatisierung

Lösung

  • Binärprotokoll — Minimaler Overhead, maximale Geschwindigkeit
  • Python / FastAPI — REST-API für Integration
  • InfluxDB — Zeitreihen-Speicherung
  • Agentic Workflow — KI plant & implementiert Module

Technische Tiefe: Binärprotokolle, SPS-Kommunikation, Echtzeit — auch hier liefert KI, wenn der Kontext stimmt.

PLCX zeigt: KI funktioniert auch in technisch anspruchsvollen Domänen. Binärprotokolle, SPS-Kommunikation, Echtzeit-Anforderungen. Der Schlüssel war Agentic Workflow — die KI hat Module geplant und implementiert, ich habe die Architektur vorgegeben und reviewt.

Sicherheit & Datenschutz

  • Keine Secrets in Prompts — API-Keys, Passwörter, Tokens
  • Firmenrichtlinien beachten — Welche Tools sind freigegeben?
  • Lokale Alternativen — Self-hosted Modelle für sensible Projekte
  • Code-Review bleibt Pflicht — KI kann Sicherheitslücken einbauen
  • DSGVO & Compliance — Wohin fließen die Daten?

Faustregel: Was du nicht in eine öffentliche E-Mail schreiben würdest, gehört nicht in einen Cloud-Prompt.

Sicherheit ist kein Blocker, aber muss bedacht werden. Enterprise-Lösungen bieten No-Training-Garantie und Audit-Logs. Für sensible Projekte: Self-hosted oder On-Premise.
